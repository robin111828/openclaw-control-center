# Code of Conduct

## Our Purpose

OpenClaw Control Center is an open-source project focused on improving usability, accessibility, and cross-device control for OpenClaw users.

We want this community to be open, practical, respectful, and welcoming to people with different backgrounds, skill levels, and perspectives.

## Our Standards

Examples of behavior that contribute to a positive environment include:

- Being respectful and constructive
- Giving clear and helpful feedback
- Focusing on the problem, not the person
- Welcoming newcomers and non-native speakers
- Sharing knowledge generously
- Accepting that reasonable people may disagree

Examples of unacceptable behavior include:

- Personal attacks, insults, or harassment
- Discriminatory or hateful language
- Trolling, intimidation, or deliberate disruption
- Publishing private information without permission
- Repeatedly derailing technical or product discussions
- Spam, malicious code, or deceptive behavior

## Project Communication Principles

When discussing features, bugs, or architecture:

- Prefer clarity over sarcasm
- Prefer evidence over assumptions
- Prefer proposals over vague complaints
- Prefer collaboration over gatekeeping

## Scope

This Code of Conduct applies to:

- GitHub issues
- Pull requests
- Discussions
- Project documentation conversations
- Community spaces directly related to this project

## Enforcement

Project maintainers may remove, edit, or reject comments, commits, code, issues, and other contributions that violate this Code of Conduct.

Maintainers may also temporarily or permanently restrict participation for repeated or severe violations.

## Reporting

If you experience or witness unacceptable behavior, please report it through the security / contact channel defined in `SECURITY.md`, or through a maintainer contact method listed in the repository.

When reporting, include as much relevant context as possible:

- What happened
- Where it happened
- When it happened
- Links or screenshots if available

## Maintainer Responsibility

Maintainers are expected to enforce this Code of Conduct fairly, consistently, and in good faith.
